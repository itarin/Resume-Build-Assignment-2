JS Promises

JavaScript is single threaded which means that functions run one at a time. This means that functions run one after the other. When one function is running it will delay whatever function comes afterwards.

At their must basic promises are like event listeners except

  • A promise can only succeed or fail once. It cannot succeed or fail twice, neither can it switch from success to failure or vice versa.
  • If a promise has succeded or failed and you later add a success/failure callback, the correct callback will be called, even though the event took place earlier.

This is very useful for success and failure in asynchrnoicity because there you are most interested in reacting to an outcome.

A promise can be:

  • Fullfilled - The action related to the promise succeeded.
  • Rejected - The action relating to the promise failed.
  • Pending - Hasn't fullfilled or rejected.
  • Settled - Has fullfilled or rejected.
